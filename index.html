How to use git

  1. Create a free account on github.com
  2. Initialise a Github repository - The easiest way to do this is on github.com by clicking "New repository"
  3. Once created click "Set up in Desktop", this will ask to to confirm opening Github desktop which you will need to have installed.
  4. After selecting a local location git will pull down all the files from Github's servers.
  5. You are now free to start writing code and adding files!
  6. Once you have made some changes and want to store them, it's time to create a commit. A commit is essentially a snapshot of your project at the time. It can be used if you ever need to come back to a point in time of your project.
  7. To create a commit, under the "Changes" tab in Github desktop, add a message to identify the changes you have made as well as an optional description with further details. Then click the "Commit" button below to create the commit.
  8. You can now refer to the "History" tab which will include a list of all commits and files changed with each.
  9. If you include configuration files with important secretive data make sure you don't add these to your repository as they will be in your history forever. This is where telling git to ignore certain files comes in handy. Create a ".gitignore" file with a list of files or folders you don't want git to track.
  10. Branching is a core concept in Git, and the entire GitHub Flow is based upon it. There's only one rule: anything in the master branch is always deployable. Because of this, it's extremely important that your new branch is created off of master when working on a feature or a fix.
  11. To create a branch click on the "Add branch" button in Github desktop. Your branch name should be descriptive (e.g. add-footer-styling), so that others can see what is being worked on.
  12. Once created you will automatically be switched to the branch where you can continue to add, change and delete files and then commit. You are now on an alternative time line and can switch between each and the project will change accordingly.
